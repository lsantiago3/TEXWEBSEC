# Task 3 – Why I Chose These Tools

I selected these tools because they match my learning style, my workflow, and the type of projects I want to build. Each tool supports me in a different part of the development process:

1. **GitHub Copilot** helps me understand code faster, generate solutions, and reduce frustration.
2. **VS Code** gives me a clean, powerful environment where I can organize files, run code, and use extensions.
3. **Git & GitHub** keep my work backed up, versioned, and easy to share or deploy.
4. **Postman / Thunder Client** make it simple to test APIs and understand how backend services communicate.
5. **Node.js + npm** let me run JavaScript projects and install packages for full‑stack development.
6. **Python** supports scripting, automation, and backend practice.
7. **Chrome DevTools** helps me debug and inspect front‑end behavior in real time.
8. **MongoDB Atlas** gives me a cloud database that’s easy to use for CRUD and backend learning.
9. **Render / Vercel** allow me to deploy projects quickly so I can see them live and share them.

Overall, these tools keep my workflow simple, efficient, and aligned with how I learn best. They reduce friction, increase speed, and help me stay focused on building real projects.
